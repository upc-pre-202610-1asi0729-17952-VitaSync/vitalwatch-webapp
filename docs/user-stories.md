Como supervisor hospitalario, quiero recibir sugerencias de reemplazo para trabajadores con turnos bloqueados, para evitar sobrecarga del resto del personal médico.

**Criterios de aceptación:**

- **Escenario 1: Generación de reemplazo sugerido**
  - **Dado** que existe un turno bloqueado.
  - **Cuando** el sistema evalúa al personal disponible.
  - **Entonces** debe sugerir trabajadores aptos para reemplazo.

- **Escenario 2: Consulta de reemplazos sugeridos**
  - **Dado** que existen reemplazos sugeridos.
  - **Cuando** el supervisor consulta un trabajador con turno bloqueado por fatiga crítica.
  - **Entonces** debe visualizar trabajadores disponibles para reemplazar dicho turno.

**Épica relacionada:** EP-04

## US-29: Consulta de trazabilidad operacional

**Título:** Consulta de trazabilidad operacional

**Descripción:**  
Como administrador hospitalario, quiero consultar las acciones preventivas registradas frente a alertas críticas de fatiga, para revisar las decisiones tomadas por supervisión hospitalaria.

**Criterios de aceptación:**

- **Escenario 1: Registro de decisión crítica**
  - **Dado** que un supervisor realiza una acción preventiva.
  - **Cuando** la acción queda registrada.
  - **Entonces** se debe guardar información del responsable, fecha y la descripción de la decisión tomada.

- **Escenario 2: Consulta de trazabilidad institucional**
  - **Dado** que existen acciones preventivas registradas.
  - **Cuando** el administrador consulta el historial de acciones.
  - **Entonces** debe visualizar las decisiones tomadas frente a alertas críticas de fatiga.

**Épica relacionada:** EP-04

## US-30: Registro e inicio de sesión de usuarios hospitalarios

**Título:** Registro e inicio de sesión de usuarios hospitalarios

**Descripción:**  
Como usuario hospitalario, quiero registrarme mediante una invitación institucional e iniciar sesión en VitalWatch, para acceder a las funcionalidades correspondientes según mi rol asignado.

**Criterios de aceptación:**

- **Escenario 1: Registro mediante invitación válida**
  - **Dado** que el usuario recibió una invitación institucional válida.
  - **Cuando** acepta la invitación y completa sus datos de registro.
  - **Entonces** el sistema debe crear su cuenta y vincularla a la cuenta hospitalaria correspondiente.

- **Escenario 2: Inicio de sesión con credenciales válidas**
  - **Dado** que el usuario posee una cuenta registrada.
  - **Cuando** ingresa sus credenciales correctamente.
  - **Entonces** el sistema debe permitir el acceso a VitalWatch según el rol institucional asignado.

- **Escenario 3: Intento de acceso inválido**
  - **Dado** que el usuario ingresa credenciales incorrectas o no pertenece a una cuenta hospitalaria.
  - **Cuando** intenta iniciar sesión.
  - **Entonces** el sistema debe denegar el acceso y mostrar un mensaje de error correspondiente.

**Épica relacionada:** EP-02
